**Q3.** [실무 적용] 직원 퇴사 시 IP + MAC 폐기만으론 부족한 이유는? 어떤 솔루션이 필요하냐?

**Q4.** [FIMS 연결] FIMS에서 FastAPI 라우터(`/api/login`)는 OSI 어느 레이어에서 동작하는 것이냐?

**Q5.** [면접 대비] `ping`은 안 되는데 `telnet`은 되는 상황, 원인을 설명하라.

**A1.** SYN → SYN+ACK → ACK (Established)

**A2.** TCP: 재전송 가능, 신뢰성 있음 → 금융/메시지. UDP: 재전송 없음, 속도 빠름 → 게임/스트리밍

**A3.** MAC 스푸핑으로 같은 MAC 복제 가능. NAC 시스템으로 MAC 충돌 탐지 + 802.1X 인증 결합해야 완결.

**A4.** L7 (응용 계층). URL 경로, HTTP 메서드, 페이로드를 처리하는 건 전부 L7.

**A5.** ICMP(ping)는 방화벽에서 차단되어 있고, 해당 포트(TCP)는 열려 있는 것. ping 안 된다고 서버가 죽은 게 아님.

## 예상 면접 질문

**Q. TCP와 UDP 차이를 설명하고, HTTPS는 어느 프로토콜 위에서 동작하는지 말하라.**
✅ 좋은 답변: "TCP는 시퀀스 번호 기반 재전송으로 신뢰성을 보장하고, UDP는 재전송 규약 없이 속도 우선입니다. HTTPS는 TCP 위에서 TLS 암호화를 얹은 프로토콜입니다."
❌ 주의: "UDP는 그냥 빠른 것"이라고만 하면 부족. "재전송 규약이 없어서 빠른 대신 유실을 감수하는 트레이드오프"까지 말해야 함.

**Q. SYN Flood 공격이 위험한 이유는?**
✅ 좋은 답변: "SYN_SENT 상태로 half-open connection을 서버에 쌓아두면 서버의 연결 테이블이 고갈되어 정상 요청을 처리 못합니다. IPS가 SYN만 반복되는 패턴을 탐지해 차단합니다."
❌ 주의: "그냥 DDoS의 일종"이라고만 하면 안 됨. half-open connection으로 인한 자원 고갈 메커니즘을 설명해야 함.

**Q. L2 스위치와 허브의 차이, PORT SECURITY가 왜 보안상 중요한지 설명하라.**
✅ 좋은 답변: "허브는 패킷을 모든 포트에 브로드캐스트하지만, L2 스위치는 MAC 테이블로 해당 포트에만 전달합니다. PORT SECURITY는 등록된 MAC 아닌 기기가 접속하면 해당 포트를 DOWN 시켜 MAC 스푸핑 및 무단 기기 연결을 차단합니다."
❌ 주의: "스위치가 더 좋다"로 끝내면 안 됨. 보안 관점에서 PORT SECURITY까지 연결해야 함.

## 연결되는 기존 개념

| 오늘 개념 | 연결되는 기존 개념 | 파일 | 어떻게 연결되나 |
|---|---|---|---|
| TCP 연결 상태 (LISTEN→ESTABLISHED) | 3-way handshake | OSI-7Layer.md | 핸드셰이크 각 단계가 LISTEN→SYN_SENT→ESTABLISHED 상태 변화 그 자체 |
| SYN_SENT 쌓임 | SYN Flood 공격 | OSI-7Layer.md | SYN만 계속 보내고 ACK 없으면 half-open connection 폭증 |
| MAC 스푸핑 → PORT SECURITY | NAC | OSI-7Layer.md | 스위치 PORT SECURITY가 1차, NAC이 2차 방어 레이어 |
| IPS가 페이로드 탐지 | 와이어샤크 패킷 구조 | OSI-7Layer.md | 오늘 와이어샤크로 본 5번줄(L7 데이터)이 IPS가 들여다보는 바로 그 부분 |
| ThroughPut | 방화벽/IPS 성능 스펙 | Security-Solutions.md | 검사 레이어 깊어질수록(L3→L7) ThroughPut 떨어짐, 고성능 장비 스펙에서 핵심 수치 |
| SSL 암호화 → 복호화 검사 | NGFW, SSL Inspection | Security-Solutions.md | NGFW가 SSL 풀고 검사 후 재암호화 = 오늘 수업의 "화이트햇이 디코딩한다"와 동일 발상 |

## 오늘 헷갈렸던 것

- **ThroughPut** — "서비스 끊기면 안 되니 많은 뭐시길 하는 거?"
  - 정리: 이론 대역폭(Bandwidth) ≠ 실제 처리량(ThroughPut). 보안 장비가 패킷 검사를 할수록 ThroughPut이 낮아짐. 장비 스펙표에서 ThroughPut 수치가 높을수록 검사 성능이 좋다는 의미.
  - 비유: 고속도로 10차선이어도 톨게이트 검문이 빡세면 실제 통과 차량은 줄어드는 것.

- **TP방화벽(Transparent Mode)** — "보충설명 필요"라고 적어둠
  - 정리: Security-Solutions.md의 "L2 브릿지 방화벽" 섹션 참고. 핵심은 IP 없이 기존 네트워크 구조 그대로 길목에 끼워 넣는 방식.
