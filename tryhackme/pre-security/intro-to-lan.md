## 한 줄 요약
LAN(Local Area Network) = 가까운 곳에 있는 기기들의 네트워크.
모양(토폴로지)을 정하고, 장비로 연결하고, 주소를 나눠주는 구조.

---

## 1. 토폴로지 (네트워크의 모양)

### 스타(Star) — 거미줄 모양 ⭐
- 모든 기기가 중앙 장비(스위치)에 각각 연결
- 지금 가장 많이 쓰는 구조
- 장점: 한 기기 고장나도 나머지 정상, 확장 쉬움
- 단점: 중앙 장비 죽으면 전체 마비, 비용 높음

### 버스(Bus) — 빨랫줄 모양
- 하나의 긴 케이블(백본)에 기기들이 매달림
- 장점: 설치 간단, 비용 저렴
- 단점: 백본 끊기면 전체 사망, 기기 많으면 느려짐

### 링(Ring) — 원형 릴레이
- 기기들이 원형으로 연결, 데이터가 원을 돌며 전달
- 장점: 케이블 적게 듦
- 단점: 중간에 하나 죽으면 전체 끊김

---

## 2. 핵심 장비

### 스위치 = 우체부
- 여러 기기를 연결하는 장비
- MAC 주소를 기억해서 정확한 기기에만 데이터 전달
- 허브와의 차이: 허브는 전체에 뿌림(비효율), 스위치는 정확히 전달

### 라우터 = 네비게이션
- 서로 다른 네트워크를 연결
- 집 안 LAN → 인터넷(WAN) 연결할 때 필요
- 데이터가 어디로 가야 하는지 경로를 잡아줌

---

## 3. 서브넷팅 (네트워크 쪼개기)

### 아파트 비유로 이해하기
- IP 주소 = 아파트 주소
- 서브넷 마스크 = 동과 호수를 나누는 기준선

### 서브넷 마스크 255.255.255.0 의미
255.255.255 . 0
네트워크 부분   기기 부분
(동까지 정해짐) (호수만 바뀜)
### 특별한 주소 세 가지
192.168.1.0   → 네트워크 주소 (동 이름표, 사용 불가)
192.168.1.1   → 디폴트 게이트웨이 (정문 경비실 = 공유기)
192.168.1.255 → 브로드캐스트 (동 전체 방송, 사용 불가)
### 호스트 수 계산
- 전체 256개 - 네트워크 주소(1개) - 브로드캐스트(1개) = 254대
- 게이트웨이(공유기)도 호스트에 포함됨

### 왜 쪼개는가?
- 회계팀(192.168.1.X)과 개발팀(192.168.2.X) 분리
- 보안 향상 + 트래픽 관리 편해짐

## 4. ARP (Address Resolution Protocol)

### 역할: IP → MAC 주소 찾기
- 데이터를 보내려면 IP만으로 부족, MAC 주소도 필요
- ARP가 "이 IP 쓰는 사람 MAC이 뭐야?" 하고 물어봄

### 작동 과정
ARP Request (방송): "192.168.1.77 누구야?"
ARP Reply (응답):   "나야! MAC은 A4:C3:F0:85:AC:2D"
ARP 캐시에 저장 → 다음엔 바로 보냄
### 보안 포인트
- ARP 스푸핑: MAC을 위장해서 다른 기기인 척 하는 공격
- 같은 IP인데 MAC이 자꾸 바뀌면 → 이상 징후!

---

## 5. DHCP (IP 자동 배정)

### 카페 번호표 비유
Discover: "IP 줄 사람?" (기기가 요청)
Offer:    "이거 써"    (서버가 IP 제안)
Request:  "그거 쓸게!" (기기가 확인)
ACK:      "확인, 쓰셈" (서버가 최종 승인)
---

## 전체 흐름
토폴로지로 모양 잡기
↓
스위치/라우터로 기기 연결
↓
서브넷으로 네트워크 나누기
↓
DHCP로 IP 자동 배정
↓
ARP로 MAC 주소 찾기
↓
데이터 주고받기 시작!
---

## 새로 알게 된 용어
| 용어 | 뜻 |
|------|-----|
| LAN | Local Area Network, 근거리 네트워크 |
| Topology | 네트워크의 물리적 모양/구조 |
| Switch | MAC 기반으로 정확히 데이터 전달하는 장비 |
| Router | 서로 다른 네트워크를 연결하는 장비 |
| Subnet | 큰 네트워크를 작게 나눈 단위 |
| Gateway | 외부 네트워크로 나가는 출입구 (공유기) |
| ARP | IP로 MAC 주소를 찾는 프로토콜 |
| DHCP | IP 주소를 자동으로 배정하는 프로토콜 |
| Host | 네트워크에 연결된 기기 |
| Broadcast | 네트워크 전체에 데이터를 보내는 것 |
