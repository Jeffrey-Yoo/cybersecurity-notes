## 새로운 구성요소 1: 로드 밸런서

### 역할: 여러 서버에 트래픽을 분산
로드 밸런서 없으면:
100만 명 → [서버 1대] → 과부하 → 다운! 💀
로드 밸런서 있으면:
100만 명 → [로드 밸런서] → 서버 1 (33만 명)
→ 서버 2 (33만 명)
→ 서버 3 (33만 명)
비유:
계산대 1개인 마트 → 줄이 엄청 길어짐
계산대 5개 + 안내 직원 → "3번 비었습니다!" → 분산
### Health Check (건강 체크)
로드 밸런서가 주기적으로:
"서버1 살아있어?" → "응!" → 정상, 트래픽 보냄
"서버2 살아있어?" → (응답 없음) → 죽었네! → 트래픽 안 보냄
= 고장난 서버를 자동으로 빼는 시스템
---

## 새로운 구성요소 2: CDN (Content Delivery Network)

### 역할: 전 세계에 정적 파일 복사본을 뿌려놓기
CDN 없으면:
한국 사용자 → 미국 서버에서 이미지 다운 → 느림 🐌
CDN 있으면:
한국 사용자 → 한국 CDN 서버에서 다운 → 빠름 ⚡
미국 사용자 → 미국 CDN 서버에서 다운 → 빠름 ⚡
비유:
CDN 없음 = 본사 창고에서만 배송 (미국→한국 일주일)
CDN 있음 = 각 나라에 물류센터 (한국 창고에서 하루)
### CDN이 저장하는 것: 정적 파일만
✅ 저장 가능: 이미지, CSS, JS, 동영상, 폰트
❌ 저장 불가: 로그인 처리, 검색 결과, DB 조회
→ 사용자마다 다르니까 원본 서버가 처리
---

## 새로운 구성요소 3: WAF (Web Application Firewall)

### 역할: HTTP 요청 내용을 보고 공격 차단 (7계층 방화벽)
일반 방화벽: IP/포트만 봄 (3~4계층)
WAF: HTTP 요청 내용까지 봄 (7계층)
정상:
GET /search?q=보안공부 → WAF: "정상" → 통과 ✅
공격:
GET /search?q=1 OR 1=1 → WAF: "SQL 인젝션!" → 차단 ⛔
POST /login 500회 연속 → WAF: "브루트포스!" → 차단 ⛔
비유:
일반 방화벽 = 건물 정문 경비원 (신분증만 확인)
WAF = 공항 보안 검색대 (가방 안 위험물까지 확인)

## 가상 호스트 (Virtual Hosts)

### 하나의 서버에서 여러 웹사이트를 동시에 운영
서버 1대 (IP: 45.33.32.156)
요청의 Host 헤더를 보고 구분:
Host: siteA.com → /var/www/siteA.com/ 폴더에서 응답
Host: siteB.com → /var/www/siteB.com/ 폴더에서 응답
Host: siteC.com → /var/www/siteC.com/ 폴더에서 응답
비유:
서버 = 하나의 건물
가상 호스트 = 건물 안 각 사무실
"A회사 찾아왔어요" → 301호로 안내
"B회사 찾아왔어요" → 402호로 안내
밖에서 보면 각각 다른 회사 사이트 같지만 같은 건물 안에 있음
### 보안 포인트
⚠️ siteA가 해킹당하면 같은 서버의 siteB, siteC도 위험!
→ 한 집 안에 세 가구가 사는 것과 같음
→ 중요한 사이트는 별도 서버를 쓰는 게 안전

## 정적 vs 동적 콘텐츠

### 정적 (Static) = 이미 만들어진 완제품
→ 누가 접속하든 항상 똑같은 내용
→ 이미지, CSS, JS, 로고, 소개 페이지
→ 서버가 파일을 꺼내서 바로 전송 (처리 없음, 빠름!)
→ CDN에 저장 가능
비유: 편의점 도시락 (이미 만들어져 있음, 꺼내서 주면 됨)
### 동적 (Dynamic) = 주문 받고 만드는 것
→ 사용자마다 다른 내용
→ "Jeff님 환영합니다", 검색 결과, 추천 상품, 장바구니
→ 서버가 요청 받고 → DB 조회 → 결과 생성 → 전송 (느림)
→ CDN에 저장 불가 (매번 다르니까)
비유: 식당 주문 요리 (주문 받아야 만들기 시작, 사람마다 다른 메뉴)
### 실제 페이지에서 보면
쿠팡 메인 페이지:
┌─────────────────────────────────────┐
│ 🔴 COUPANG 로고              🔍    │ ← 정적 (항상 같음)
├─────────────────────────────────────┤
│ 카테고리: 전자제품 | 패션 | 식품     │ ← 정적 (메뉴 같음)
├─────────────────────────────────────┤
│ "Jeff님을 위한 추천 상품"           │ ← 동적! (내 데이터 기반)
│ ┌─────┐ ┌─────┐ ┌─────┐          │ ← 동적! (사람마다 다름)
├─────────────────────────────────────┤
│ © 2026 Coupang. All rights reserved│ ← 정적 (항상 같음)
└─────────────────────────────────────┘
정적 → CDN이 처리 (빠르고 서버 부담 줄임)
동적 → 원본 서버가 처리 (로드 밸런서로 분산)
### 백엔드 코드를 사용자가 볼 수 있어?
절대 못 봄 (Nay!)
프론트엔드: 브라우저에서 실행 → 소스 보기 가능
백엔드: 서버에서 실행 → 결과만 전달, 코드는 안 보임

## 전체 9단계 — 모든 걸 합치면

### tryhackme.com에 접속할 때 일어나는 일
브라우저에 tryhackme.com 입력
↓
DNS에서 IP 주소 확인
↓
올바른 IP인지 확인 (캐시 or 조회)
↓
WAF가 요청 검사 (공격인지 확인)
↓
로드 밸런서가 적절한 서버로 분배
↓
웹서버가 요청 수신 (Virtual Host로 사이트 구분)
↓
백엔드에서 동적 콘텐츠 처리 (DB 조회 등)
↓
서버가 HTML/CSS/JS 응답
↓
브라우저가 화면에 렌더링
### 식당 배달 주문 비유
"맛있는 피자집 전화번호 뭐지?"     (URL 입력)
전화번호부에서 번호 찾기            (DNS 조회)
"이 번호 맞나?" 확인               (IP 확인)
직원이 "장난 전화 아니지?" 확인     (WAF)
"3번 주방이 여유있으니 거기서"      (로드 밸런서)
3번 주방 요리사가 주문 받음         (웹서버)
재료 꺼내서 요리 시작              (백엔드 + DB)
완성된 피자를 배달 보냄             (HTTP 응답)
집에서 피자 받아서 먹음!            (브라우저 렌더링)
### 각 단계가 어느 강의에서 배운 건지
1~3단계: DNS in Detail
→ 도메인을 IP로 바꾸는 과정
4단계: Extending Your Network + 이번 강의
→ WAF = 7계층 방화벽
5단계: 이번 강의 (새로 배움)
→ 로드 밸런서 + Health Check
6단계: HTTP in Detail + 이번 강의
→ Host 헤더로 사이트 구분 (Virtual Hosts)
7단계: How Websites Work
→ 백엔드가 DB 조회해서 동적 콘텐츠 생성
8단계: HTTP in Detail + Packets & Frames
→ HTTP 응답 + TCP 전송
9단계: How Websites Work
→ HTML + CSS + JS 조합해서 화면 표시
### 헷갈리기 쉬운 순서
DNS → WAF → 로드 밸런서 (이 순서 중요!)
DNS가 먼저: IP 모르면 서버에 도착조차 못함
WAF가 다음: 서버 도착 후 공격 걸러냄
로드 밸런서가 마지막: 정상 요청만 서버에 분배
### 암기 팁
"부 디 와 로 웹 백 응 렌"
브라우저 → DNS → WAF → 로드밸런서 → 웹서버 → 백엔드 → 응답 → 렌더링
또는:
"찾고 → 검사하고 → 나누고 → 받고 → 처리하고 → 보내고 → 보여주고"

## 새로 알게 된 용어
| 용어 | 뜻 |
|------|-----|
| Load Balancer | 여러 서버에 트래픽을 분산시키는 장치 |
| Health Check | 로드 밸런서가 서버 생존 여부를 확인하는 것 |
| CDN | Content Delivery Network, 전 세계에 정적 파일 분산 저장 |
| WAF | Web Application Firewall, HTTP 내용을 보고 공격 차단 |
| Virtual Hosts | 하나의 서버에서 여러 사이트를 운영하는 기능 |
| Static Content | 정적 콘텐츠, 변하지 않는 파일 (이미지, CSS, JS) |
| Dynamic Content | 동적 콘텐츠, 사용자마다 다른 내용 (검색 결과, 추천) |
| Rendering | 브라우저가 HTML/CSS/JS를 조합해서 화면에 표시 |
