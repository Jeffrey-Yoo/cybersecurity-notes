# 2026-06-09 — 실습 랩 환경 구축 시작 (Workstation Pro · ESXi)

과정2 6일차 오후. 그동안 책으로만 보던 "보안 장비 배치 / 탭 / DLP / 가상화" 흐름을 **내 PC 한 대 위에 가상으로 직접 세워보는** 실습의 첫날. 오늘은 토대(Workstation Pro → ESXi)까지 깔고, 만들 목표 구성과 내 홈 네트워크를 설계했다.

> FIMS 연결: 과정1에선 VirtualBox 내부 네트워크로 세그멘테이션했다. 이번엔 한 단계 위 — **ESXi(데이터센터급 하이퍼바이저) + pfSense(방화벽)**로 실무에 더 가까운 구조를 직접 만든다.

---

## 오늘 한 것

```
[완료] VMware Workstation Pro 설치   ← Broadcom 배포본
[완료] 그 위에 ESXi 8.0 설치          ← Workstation 위에 ESXi를 올리는 "중첩 가상화(nested)"
```

- **중첩 가상화 구조**: Workstation Pro(타입2 HV) 안에서 ESXi(타입1 HV)를 돌린다. ESXi가 "데이터센터의 베어메탈 하이퍼바이저" 역할을 흉내 내고, 그 위에 pfSense·Ubuntu·Kubuntu를 게스트로 띄울 예정.

> 비유: Workstation Pro라는 **건물(PC)** 안에 ESXi라는 **작은 데이터센터**를 통째로 넣고, 그 데이터센터 안에 방화벽·서버들을 입주시키는 그림.

⚠️ **음차 교정** (수업 노트에 잘못 적힌 것)
- "Wmware" → **VMware**
- ESXi 버전 → **8.0**
- VMware 인수사 → **Broadcom**


## 만들려는 목표 랩 구성 (클래스 예시)

```
노트북 (172.30.1.15)
   │  VMware Host Client 접속 (브라우저로 ESXi 관리)
   ▼
┌──────────────── ESXi 호스트 ────────────────┐
│        pfSense FW                            │
│   NIC1 = WAN (브릿지)  ── 공유기 쪽(외부)      │
│   NIC2 = 내부 네트워크 ── 게스트들이 붙는 내부  │
└──────────────────────────────────────────────┘
```

- pfSense에 **NIC 2장** — NIC1=WAN(브릿지로 공유기에 직결), NIC2=내부망. 1차 FIMS 때 방화벽이 인터넷↔내부를 가르던 구조를 가상으로 재현하는 것.
- 노트북에서 **VMware Host Client**(브라우저 기반 ESXi 관리 콘솔)로 ESXi에 접속해 게스트를 관리한다.

> pfSense NIC1/NIC2 = 방화벽 인터페이스 구간 분리. 이론노트 `../IT-Infra-Secu/Architecture.md` "방화벽 디자인 A/B/C/D 코스"를 직접 손으로 만지는 셈.

## 내가 실제로 구성한 홈 네트워크 환경 (Excalidraw 기반)

클래스 예시(172.30.1.x)를 **내 집 네트워크(192.168.1.0/24)**에 맞춰 직접 그린 실제 구성:

```
                       인터넷
                         │ 공인 211.x.x.x
                   ┌─────┴─────┐
                   │   공유기   │  내부 G/W 192.168.1.1
                   └─┬───┬───┬─┘
                     │   │   └──────── 다른 무선 장치 (무선 직결)
         내 본컴     │   │   서브컴(LAB)
      192.168.1.83 ──┘   └── 192.168.1.63
            │                      │  └ 위에 ESXi 올려 가상 인프라 구동
            │   NFS 192.168.1.220  │
            └── (본컴→서버컴 용량 추가) ─┘
                                   ▼
        ┌──────── 서브컴(LAB) / ESXi 192.168.1.200 ────────┐
        │  Management Network                              │
        │  pfSense ── NIC1(WAN) / NIC2(내부)               │
        └──────────────────────────────────────────────────┘
```

| 구성요소 | 주소 | 역할 |
|---|---|---|
| 공유기 | 공인 211.x.x.x / 내부 G/W 192.168.1.1 | 집 전체 인터넷 출입구 + 내부망 게이트웨이 |
| 내 본컴 | 192.168.1.83 | 메인 PC. NFS로 서브컴(LAB)에 저장공간 제공 |
| 서브컴(LAB) | 192.168.1.63 | 실습 전용 머신. 위에 ESXi 올림 |
| ESXi | 192.168.1.200 | 서브컴 위 하이퍼바이저 (관리: Management Network) |
| NFS | 192.168.1.220 | 본컴 → 서버컴(LAB) **용량 추가**용 네트워크 스토리지 |
| 다른 무선 장치 | (무선) | 공유기에 무선으로 붙는 단말들 |

- **NFS로 용량을 빌려준다** — 실습용 서브컴의 디스크가 모자라서, 본컴의 저장공간을 NFS(192.168.1.220)로 마운트해 **서버컴 용량을 늘리는** 구성. (NFS = Linux 전용 네트워크 파일공유 → `../IT-Infra-Secu/Architecture.md` "파일 공유 서버 아키텍처")

## ⚠️ 짚어둘 것 / 앞으로 막힐 만한 곳

- **무선 음영지역과 직결** — 이 그림의 "다른 무선 장치"는 공유기에 무선으로 직접 붙어 길목을 안 거친다. 6-9일 오전에 배운 IDS 음영지역(`../IT-Infra-Secu/Security-Solutions.md` "TAP은 길목이 있어야 한다")이 **내 집 구성에도 그대로 적용**된다 → 나중에 SPAN/TAP을 걸어도 무선 단말 트래픽은 안 잡힐 것.
- **중첩 가상화 성능** — Workstation 위 ESXi 위 게스트라 자원을 많이 먹는다. 서브컴 CPU 가상화 지원(VT-x/AMD-V)·중첩 가상화 옵션이 켜져 있어야 게스트가 뜬다.
- **ESXi NIC 호환성** — ESXi는 지원 NIC 드라이버가 까다로운 편. 중첩 환경에선 Workstation이 가상 NIC를 제공하므로 보통 무난하지만, 실패 시 어댑터 타입(E1000 vs VMXNET3)부터 의심.
- **WAN 브릿지 = 공유기에서 IP를 받는 구조** — pfSense WAN(NIC1)이 브릿지면 공유기 DHCP에서 192.168.1.x 한 자리를 잡는다. 위 IP 표와 충돌하지 않게 고정/예약 관리 필요.
