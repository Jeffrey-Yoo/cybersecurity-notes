# IT 인프라의 종류, 기능의 이해

## 1. IT 보안이란?

민감 정보를 지키기 위해 시스템, 네트워크, 데이터를 공격과 무단 접근으로부터 보호한다.

> Red Hat, Cisco 두 회사의 예시를 들어 IT 보안의 근본적인 핵심을 설명

---

## 2. IT 보안이 필요한 이유

- 디지털 자산과 네트워크 장비 보호의 필요성 증가
- 침해 발생 시 훨씬 더 많은 비용 발생
- 기업 존폐 자체가 위험해질 가능성
  - 내부 직원 소행으로 정보를 팔아버리는 사례 존재

> IT 보안팀은 사고를 해결 혹은 해결 후 대응 계획을 위험 관리 툴을 사용해 상황을 통제

---

## 3. IT 보안과 정보 보안(InfoSec)의 차이점

| 구분 | 정의 |
|------|------|
| **정보 보안 (InfoSec)** | 민감한 기업 정보를 침입으로부터 보호할 프로세스와 툴 — 정책 영역 |
| **IT 보안** | 컴퓨터 네트워크를 통해 디지털 데이터를 보안하는 작업 — 기술 영역 (예: 방화벽, Nmap) |

> 서로 유기적으로 연결되어 보안이 설계됨

---

## 4. IT 보안을 위협하는 요인

### 악성코드 (Malware / Malicious Software)
- Ransomware
- Spyware
- Virus

> 위와 같은 위협에 대한 보안 지침 및 예방안을 구축하는 것이 핵심

### 내부 위협
- 직원
- 협력사 직원

> 해커보다 침투하기 수월 → 해커가 수행하는 어려운 작업을 보다 쉽게 수행 가능

---

## 5. 정보 보호의 목적

1. **사업 보호** — 정보 유출, 기밀 및 영업 비밀 유출 방지
2. **사업 개시** — 온라인 교육사업 등을 위한 정보보안 정책 및 처리 시스템 구축
3. **고객 가치 창출** — 제품 서비스 애플리케이션 보안 정책 수립
4. **팀의 성과 사수 및 리스크 감경** — 악의적 혹은 실수로 인한 처벌 방지

---

## 6. 보안 분야

각각의 보안 특수 분야 및 기술에 따라 파트가 나뉨 (네트워크 보안, 애플리케이션 보안, 클라우드 보안 등)

---

# IT 인프라 (IT Infrastructure)

포괄적 의미를 보유하는 정보 기술 구성요소 집합 (컴퓨터, 네트워크, 하드웨어, 시설 등)

---

## IT 인프라 구성 요소

### 1. 하드웨어
- 서버 및 스토리지(저장소)
- **라우터** : 네트워크가 길을 찾아가도록 하는 장치
- **스위치 (L2, L3, L4)** : 라우터보다 작은 형태로 길을 찾아주는 장치
- 데이터 센터 / 회사 내 전산실
- 개인용 컴퓨터

### 2. 소프트웨어
- 웹서버
- 콘텐츠 관리 시스템 (CMS)
- Linux OS
- 기업용 애플리케이션

### 3. 네트워킹
- 방화벽, 라우터, **패치 패널** 등 하드웨어 포함

---

## IT 인프라 유형

| 유형 | 설명 |
|------|------|
| **컨버지드 (On-Premises)** | 전통적 인프라 — 필요 요소 전부를 시설에서 직접 구비/관리. 비용·전력·물리 공간 필요 |
| **클라우드 (Cloud)** | 클라우드 컴퓨팅 리소스를 공유/임대하는 형태 |
| **하이퍼 컨버지드 (HCI)** | 단일 인터페이스에서 컴퓨터·네트워크·데이터 리소스를 통합 관리. 4대 컴퓨터 자원을 1개처럼 사용 = 슈퍼컴퓨터 개념 |

---

## IT 인프라 관리 영역

IT 인프라 관리는 IT 리소스, 시스템, 플랫폼, 인력, 환경을 제어한다.

```
인프라 관리
│
├── 기반 환경 관리
│   ├── OS 관리             → 바닥 공사
│   ├── 클라우드 관리        → 외부 임대 공간 관리
│   └── 가상화 관리 (HCI)   → 공간 효율화
│
├── 운영 효율화
│   ├── IT 자동화                  → 반복 작업 로봇화
│   ├── 컨테이너 오케스트레이션     → 앱 배달 물류 자동화
│   └── 설정 관리                  → 전국 매장 레시피 통일
│
└── 보안/거버넌스
    ├── API 관리    → 앱 간 대화 창구 통제
    ├── 리스크 관리 → 위험 사전 예측 및 대응
    └── 데이터 관리 → 데이터 위치 및 접근 추적
```
