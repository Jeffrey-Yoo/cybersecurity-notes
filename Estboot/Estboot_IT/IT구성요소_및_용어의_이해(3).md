# IT 구성요소 및 용어의 이해 (3) — 망분리 · 망연계 · DRM

## 📝 망분리

내부 정보 유출을 차단하기 위한 목적성을 가진 내부/외부망 분리

- 법적 의무 (전자금융감독 규정, 정보통신망법 등)로 일부 강제 시행
- 개인 정보 및 중요 전자 자산 보호를 위한 설계
- 폐쇄망을 따로 설계해 인가된 인원이 물리적으로만 접근할 수 있는 구역을 설계

### 망 구성 3종류

| 망 | 설명 |
|---|---|
| **업무망** | 사내 내부 시스템 접근용. 인터넷 차단 |
| **인터넷망** | 외부 인터넷 접근용. 내부 시스템 접근 불가 |
| **폐쇄망** | 인터넷과 완전히 단절된 망. 군사/원자력 등 최고 보안 등급 구역에 사용. 인가된 인원만 물리적으로 접근 가능 |

---

## 📝 물리적 망분리

- 업무망과 인터넷망을 물리 하드웨어로 분리
- 여러 조건 (보안 등급, 재정 등)에 의해 설계
- 전용 하드웨어 장비(고가)로 망연계가 이루어져 보안 측면에서 이점이 많음

### 물리적 망연계 특징

- **Non TCP/IP** 방식으로 인터넷 차단
- **DLP** (Data Loss Prevention) — 데이터 유출 방지 솔루션 포함. 탐지뿐만 아니라 차단까지 수행
- 관리자의 승인 요청 필요
- 물리적인 Switch (KVM) 존재 → 1개의 물리 하드웨어에서 두 망 전환 가능

---

## 📝 망연계

보안 구역과 비보안 구역 간의 데이터 이동 구간 (연계 구간)

> 연계 구간 경계를 **망연계 게이트웨이** (Ninja Gate)라고도 부름

### 물리적 망분리에서 자료 연계 절차

```
1. 자료 전송 로그인
2. 하드웨어 정보 일치 확인
3. 데이터 파일 전송 시도
4. 내부 중계 서버 업로드
5. 승인 요청 → 외부 관리자
6. 검토 및 승인 → 전송
```

> 업로드/다운로드 검사 엔진이 동일해야 함

---

## 📝 논리적 망분리 — 서버 기반 (SBC)

**SBC : Server Based Computing**

- 가상화 기술로 서버를 분리 (VLAN)
- 데이터는 기본적으로 읽기 전용
- 물리적 망분리와 다르게 L3 (라우터, 백본)에서 각 망으로 네트워크 분리
- HCI로 설계된 서버에서 가상 데스크탑 화면을 송출받아 외부망과 연결 (VDI)
- 내 PC는 내부망에 연결되어 있고 VDI로 외부망에만 가상 연결

> 서버의 소프트웨어를 통해 망연계가 이루어짐

---

## 📝 논리적 망분리 — 클라이언트 기반 (CBC)

**CBC : Client Based Computing**

- 사용자 PC에 가상화 프로그램을 설치해 클라이언트로서 망 구분
- 내 PC 자체에서 직접 사용
- 이식성이 좋음
- 보안 측면에서 SBC보다 취약
- 가격 이점이 있지만 각 하드웨어의 성능이 좋아야 함

> 사용자 PC 소프트웨어 기반으로 망연계가 이루어짐

---

## 📝 망분리 방식 비교

| 구분 | 물리적 | 논리적 SBC | 논리적 CBC |
|---|---|---|---|
| 망연계 위치 | 전용 하드웨어 장비 | 서버 소프트웨어 | 사용자 PC 소프트웨어 |
| 보안 수준 | 최상 | 상 | 중 |
| 비용 | 높음 | 중간 | 낮음 |
| 사용자 PC 성능 | 무관 | 낮아도 됨 | 높아야 함 |

## 📝 Software

### DRM (Digital Rights Management)

무단 디지털 콘텐츠 공급/제공을 막는 기술 서비스

- Agent를 통해 하드웨어 파일을 자동 암호화
- 각 라이선스 키가 존재
  - 수신자는 라이선스 키를 제공받아야 열람 가능
  - 복호화 후 파일 제공 가능
- 하드웨어 제어권 통제 → 관리자가 하드웨어 제어 및 통제 가능
- 기업 보안 인프라의 핵심 구성요소. 서버 안정적 운영과 권한 관리가 함께 따라와야 함
- 같은 DRM 솔루션이라도 **라이선스 서버가 다르면 키가 달라서 타 기업과 문서 공유 불가**

## 💭 오늘의 회고

### 배운 점
- 무결성과 기밀성을 위한 망분리가 어떻게 이루어지는지 구조적 이해
- 기업 내 악성코드를 원천 차단하기 위한 아키텍처 이해
- 목적과 실정에 따라 사용되는 망의 종류 이해
- 여러 망분리 체계 이해

### 어려운 점 / 개선할 점
- 구조적으로는 이해가 됐으나 실질적인 실습이 필요
- 망분리, 연계 과정에서 세부적으로 어떤 기능이 작용해 무결성이 확보되는지 심도 깊은 이해 필요

## 📚 알게 된 IT 용어

| 용어 | 설명 |
|---|---|
| **Access Control** | 접근 통제. 필로티 구조로 외/내부인 공간 분리 가능 |
| **RBAC (Role Based Access Control)** | 역할 기반 접근 제어 |
| **CS Program** | 고객관리용 프로그램 |
| **망연계 게이트웨이 (Ninja Gate)** | 연계 구간 경계 |
| **DLP (Data Loss Prevention)** | 데이터 유출 방지 솔루션. 탐지 + 차단 수행 |
| **PMS (Patch Management System)** | 패치 관리 시스템 |
| **IPS (Intrusion Prevention System)** | 침입 방지 시스템. OSI 7계층에서 동작. DDoS 탐지/완화 기능 포함 (전담 장비는 별도 존재) |
| **Local (Window)** | 현재 내 컴퓨터 자체 |
| **VDI (Virtual Desktop Infrastructure)** | 가상 데스크탑 환경 |
| **SBC (Server Based Computing)** | 논리적 망분리 서버 기반 |
| **CBC (Client Based Computing)** | 논리적 망분리 클라이언트 기반 |
| **MDM (Mobile Device Management)** | 모바일 제어 통제 관리 시스템 |
| **DRM (Digital Rights Management)** | 무단 디지털 콘텐츠 통제 기술 서비스 |
