# 데이터 센터 및 IT 구성요소의 이해

## 📝 데이터 센터 (IDC / Internet Data Center)

인터넷과 연결된 데이터를 모아두는 시설

| 유형 | 설명 |
|------|------|
| **Public** | 누구에게나 열림 (Hosting) |
| **Public and Owned** | 고객 및 내부 사용 (Hybrid) |
| **Owned** | 내부 사용 Only |

---

### 보안을 위한 통제 관리

- **관리자 출입 통제**
- **방문자(기술 직원) 출입 통제**
  - 방문자가 현장에서 로깅할 수 없도록 통제 및 감시
  - USB 등 반출입 통제 (CD 사용 = 데이터 입력 방지)
- **서버 반입/반출**
  - 새 장비인지, 이전에 패치가 이루어지지 않았는지 확인 = 무결성 확보
- **Data 반출**
  - Disk 불량 교체 시 → 폐기, 삭제 or 복제
  - **Degaussing**으로 영구 폐기
    - 자성을 이용한 저장장치(HDD)에만 적용됨
    - SSD/USB는 자성 기반이 아니므로 물리 파쇄 또는 암호화 삭제 방법 별도 필요

---

## 📝 인터넷 회선

### 1. ISP (Internet Service Provider)
- 인터넷을 개인이나 기업에게 제공하는 회사
- 1Gbps ~ 필요 용량 = 메인 회선 / 보안 장비
- 보안 측면에서 회선을 두 개 써 가용성 확보 가능 (데이터센터 운영)

> IT 업계에선 ISP를 '회사'보다 '회선'의 의미로 읽는 경우가 많음

---

### 2. HCX (Hyper Connect Experience) = KT 자사 직통신

```
클라우드(CSP) ↔ 데이터센터(중계)(IDC) ↔ 내 회사 데이터센터
```

- 일반 네트워크를 통한 데이터 송수신은 기밀성, 비용, 속도 측면에서 이점 없음
- HCX 물리적 케이블 사용으로 위 단점을 모두 보완
- 유선 전화기처럼 스니핑이 불가한 구조

---

### 3. 국제망 (Submarine Cable) = 해저 케이블
- 실제 해저케이블로 각 나라끼리 물리적으로 연결된 국제 통신망

---

### 4. MSPP (Multi-Service Provisioning Platform)
- 속도 제한이 높은 직통신 전용선
- HCX와 비슷하지만 **자사끼리** 보안 및 효율성을 위한 데이터 전송 방법으로 사용
- 범용이기 때문에 유연한 성질을 가짐
- 여러 통신 신호를 하나로 묶어 전송

---

### 5. DWDM (Dense Wavelength Division Multiplex)
- 광케이블 하나로 여러 파장을 동시 전송
- 속도 측면에서 매우 유리

---

### 6. ROADM (Re-configurable Optical Add-Drop Multiplexer)
- DWDM의 확장
- DWDM을 유연하게 원격 제어 가능

---

## 📝 보안 시스템

### 1. 방화벽 (F/W)
- **UTM** : 여러 보안 장치의 역할을 하나의 장치에서 수행 가능
  - 비용 절약에 유리하나 보안적으로는 각 장치를 분리 배치하는 것이 유리

### 2. DDoS 방어 장비
- DDoS (Distributed Denial of Service) 공격을 방어하기 위한 전용 장비

### 3. SSL Inspection
- 전송 계층에서 암호화된 정보를 **복호화(Decryption)** 하여 평문화한 뒤 저장
- 일반 사용자의 데이터가 저장된다는 단점이 있으나 기밀성만 유지된다면 악성코드 유입 차단에 유리

### 4. WAF (Web Application Firewall)
- 웹 공격 특화 악성코드 방지 장치

### 5. WIPS (Wireless Intrusion Prevention System)
- 비인가 장치를 찾아 데이터센터 및 보안이 필요한 곳의 네트워크 차단
- 인가된 Wi-Fi만 접근 허용
- **Jamming** 기술 사용 → 강한 전파로 전자기기 무력화 → SSID 식별 불가

## 📝 네트워크 계층 구조

```
Router       → 네트워크를 모아 길을 찾아주는 역할
  ↓
Backbone     → 핵심 고속 네트워크 회선
  ↓
L4           → L3, L2 기능을 모두 포함 가능
  ↓
L3
  ↓
L2
```

## 💭 오늘의 회고

### 배운 점
- 실제 데이터센터에서 어떤 회선을 통해 어떤 네트워크 절차를 거쳐 송수신 되는지
- 가용성과 기밀성을 위한 다양한 통신 방법
- 데이터센터 내부에서 기밀성과 가용성을 위해 장비가 어떻게 관리되는지
- 통신기업들이 수많은 보안 장치로 사용자 정보를 보호하고 있다는 사실

### 어려운 점 / 개선할 점
- 이전에 공부했던 네트워크 내용과 접목하여 이해하는 데 어려움
- 여러 보안 기사를 분석해 배운 내용과 접목하여 이해 확장 필요

## 📚 알게 된 IT 용어

| 용어 | 설명 |
|------|------|
| **UPS** | 보조 배터리 역할 (무정전 전원 공급 장치) |
| **NTP** | Network Time Protocol — UTC 동기화 시 필요 |
| **Parameter** | 매개 변수 — 동작 및 특성 제어 변수 |
| **Hash** | MD5 등 알고리즘을 통해 나온 고유 값. 두 값이 같다면 무결성 증명. 현재는 보안상 **SHA-256** 권장 |
| **MD5** | Hash 값을 생성해주는 알고리즘. 현재는 취약점 발견으로 SHA-256으로 대체 |
| **Degaussing** | 반자기화 — 자성을 잃게 해 HDD 데이터를 영구 폐기. SSD에는 적용 불가 |
| **미러링 (Mirroring)** | 동기화 작업 (가용성 확보) |
| **핫스왑 (Hot Swap)** | 서버 전원 유지 상태에서 물리적으로 하드를 교체하는 행위 |
| **오토 리빌드 (Auto-Rebuild)** | 핫스왑 후 새 하드에 데이터를 동기화하는 작업 (RAID 개념) |
| **I/O** | Input / Output — 읽고 쓰는 동작 |
| **NIC** | Network Interface Card — 랜카드 |
| **Jamming** | 의도적으로 강한 전파를 보내 전자기기 신호 방해 |
| **SSID** | Service Set Identifier — Wi-Fi 이름 |
| **IDS** | Intrusion Detection System — 침입 탐지 시스템 |
| **IPS** | Intrusion Prevention System — 침입 방지 시스템 |
