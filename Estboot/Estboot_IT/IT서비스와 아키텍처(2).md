# 오늘의 TIL 보충 설명 — IP / 서브넷 / AWS / 네트워크 용어

## 1. IP 주소란 무엇인가

IP 주소는 네트워크에서 각 장치를 구별하는 주소다. 우편 주소랑 똑같은 개념이다.

형태: `192.168.56.102`
- 4개의 숫자 덩어리로 구성
- 각 덩어리는 0~255 사이의 숫자
- 각 덩어리를 **옥텟(Octet)** 이라고 부름
- 4개 덩어리 = 32비트 = IPv4

---

## 2. 서브넷마스크란 무엇인가

IP 주소는 두 부분으로 나뉜다.

```
[네트워크 주소] + [호스트 주소]
(어느 동네인가)   (동네 안 몇 번지인가)
```

서브넷마스크는 이 두 부분의 경계선을 알려주는 것이다.

예시: `192.168.56.102 / 255.255.255.0`

```
255.255.255 → 네트워크 주소 (고정, 변하지 않음)
.0          → 호스트 주소 (여기서 장치 구별)
```

→ `192.168.56` 까지는 같은 동네, 마지막 숫자(0~255)로 같은 동네 안의 장치를 구별

---

## 3. C클래스 — 서브넷 이해

C클래스: `255.255.255.0` (또는 /24)

마지막 자리가 0~255이니까 256개인데 실제로 쓸 수 있는 건 **254개**다.

```
192.168.56.0   → 네트워크 주소 (예약됨, 사용 불가)
192.168.56.1   → 사용 가능 (보통 게이트웨이/라우터)
192.168.56.2   ~ 192.168.56.254 → 사용 가능
192.168.56.255 → 브로드캐스트 주소 (예약됨, 사용 불가)
```

> **브로드캐스트란?** 같은 네트워크 안의 모든 장치에게 동시에 신호를 보내는 것

### 클래스별 비교

| 클래스 | 서브넷마스크 | 수용 가능 장치 | 용도 |
|---|---|---|---|
| **A클래스** | 255.0.0.0 (/8) | 약 1600만개 | 대형 기업/국가 |
| **B클래스** | 255.255.0.0 (/16) | 약 65000개 | 중형 기업 |
| **C클래스** | 255.255.255.0 (/24) | 254개 | 소형 네트워크 |

## 4. SNAT와 DNAT

**NAT(Network Address Translation)** = IP 주소를 변환하는 기술

> 내부망의 PC들은 사설 IP(192.168.x.x)를 쓴다. 사설 IP는 인터넷에서 쓸 수 없으므로 인터넷으로 나갈 때 공인 IP로 바꿔줘야 한다.

### SNAT (Source NAT) — 출발지 주소 변환
```
내 PC (192.168.1.10) → 인터넷 접속 시 → 공인 IP (123.456.789.0)로 변환
```
→ 내부에서 외부로 나가는 트래픽에 사용

### DNAT (Destination NAT) — 목적지 주소 변환
```
외부에서 서버 접속 시 → 공인 IP → 내부 서버 사설 IP로 연결
```
→ 포트 포워딩이 대표적인 DNAT 사례

| 구분 | SNAT | DNAT |
|---|---|---|
| 방향 | 내부 → 외부 | 외부 → 내부 |
| 변환 대상 | 출발지 IP | 목적지 IP |
| 비유 | 편지 발신인 주소를 회사 주소로 바꾸는 것 | 대표 번호로 전화 오면 담당자에게 연결 |

## 5. Load Balancer (로드밸런서)

### 왜 필요한가

서버 한 대로 수백만 명의 요청을 처리하면 서버가 죽는다.

```
사용자 100만 명
      ↓ 전부 서버 1대로 몰림
서버 1대 → 과부하 → 서비스 중단 ❌
```

로드밸런서는 이 트래픽을 여러 서버에 나눠주는 장치다.

```
사용자 100만 명
      ↓
[ Load Balancer ]
  ↙     ↓     ↘
서버1  서버2  서버3  → 각각 33만 명씩 처리 ✅
```

### 로드밸런서 동작 방식

```
사용자 요청 도착
      ↓
로드밸런서가 서버 상태 확인
      ↓
살아있는 서버 중 가장 여유 있는 곳으로 전달
      ↓
서버가 응답 → 사용자에게 전달
```

서버 하나가 죽어도 로드밸런서가 살아있는 서버로만 보내기 때문에 서비스가 유지된다.

### 분산 알고리즘 종류

| 방식 | 설명 | 특징 |
|---|---|---|
| **라운드로빈** | 서버에 순서대로 하나씩 배분 | 가장 단순. 균등 배분 |
| **최소 연결** | 현재 연결이 가장 적은 서버로 | 서버 부하 기준 배분 |
| **IP 해시** | 같은 사용자는 항상 같은 서버로 | 세션 유지 필요 시 |
| **가중치** | 성능 좋은 서버에 더 많이 배분 | 서버 사양이 다를 때 |

### OSI 계층별 로드밸런서

| 구분 | 계층 | 기준 | 예시 |
|---|---|---|---|
| **L4 스위치** | 4계층 (전송) | IP + 포트 번호 | AWS NLB |
| **L7 스위치** | 7계층 (응용) | URL, 쿠키, 헤더 | AWS ALB |

> **L4 vs L7 차이**
> L4는 "어디서 왔냐(IP/포트)"만 보고 분배
> L7은 "뭘 요청하냐(URL 내용)"를 보고 분배
> 예: `/api` 요청 → API 서버로, `/image` 요청 → 이미지 서버로 (L7만 가능)

### 백화점 아키텍처에서의 위치

```
사용자 (인터넷)
      ↓
DDoS 방어 장비
      ↓
방화벽
      ↓
[ Load Balancer (L4/L7) ]  ← 여기
  ↙         ↓         ↘
웹서버1    웹서버2    웹서버3  (DMZ 구간)
      ↓
WAS (내부망)
      ↓
DB
```

### 클라우드와 연동 시 (오늘 수업 내용 연결)

```
메인 데이터센터 서버
      ↓
[ Load Balancer ]  ← 트래픽 분산
  ↙         ↘
온프레미스 서버   AWS 클라우드 서버
```

트래픽이 몰릴 때(명절, 세일 기간) 클라우드 서버로 자동 분산 → **버스팅** 개념

## 6. AWS 관련 용어

### Amazon Redshift
- 아마존 클라우드 기반 **DW(데이터 웨어하우스)** 서비스
- 대용량 데이터 분석에 특화. 수백 TB ~ PB 규모도 처리 가능
- 일반 DB(MySQL)는 실시간 처리용, Redshift는 **분석용**

### S3 (Simple Storage Service)
- 아마존 클라우드 오브젝트 스토리지
- 파일, 이미지, 영상, 백업 데이터 저장. 용량 제한 없음

| 스토리지 종류 | 설명 |
|---|---|
| DAS | 서버에 직접 붙인 저장장치 |
| NAS | 네트워크로 연결된 공유 저장소 |
| SAN | 고속 전용 네트워크 스토리지 |
| **S3** | 클라우드 기반 무제한 저장소 |

## 7. JDBC Connector

**JDBC = Java Database Connectivity**

자바 기반 앱이 DB에 접속할 때 쓰는 표준 인터페이스. 앱과 DB 사이의 연결 다리 역할.

```
앱(Java) → JDBC Connector → DB (MySQL / Oracle / Redshift 등)
```

오늘 실습에서는 DW 솔루션(Redshift)에 처음 연결할 때 JDBC Connector를 통해 통신을 연결한 것.

## 8. BI (Business Intelligence)

DW에서 정제된 데이터를 시각화하고 분석해서 경영 의사결정에 활용하는 것.

```
원시 데이터 → DW(정제/가공) → BI 도구(시각화) → 경영진 의사결정
```

**대표 BI 도구:** Tableau, Power BI, AWS QuickSight

**백화점 예시:**
```
고객 구매 데이터 → Redshift(정제) → BI 대시보드
→ "30대 여성이 화요일 오후에 가장 많이 구매한다" 분석
→ 해당 시간대 타겟 마케팅 이벤트 기획
```

## 9. Disk 복제와 DWDM — 혼동 주의

오늘 노트에서 혼동된 부분 정정:

| 구분 | 설명 | 목적 |
|---|---|---|
| **Disk 복제** | 데이터를 다른 디스크에 동일하게 복사 (RAID 1 미러링) | 가용성 확보 |
| **DWDM** | 광케이블로 대용량 데이터를 빠르게 전송하는 회선 기술 | 전송 속도 향상 |

> DWDM은 복제된 데이터를 빠르게 **보내는 수단**이고, Disk 복제는 데이터를 동일하게 **유지하는 작업**이다.

## 10. Backup 종류

| 방식 | 설명 | 장점 | 단점 |
|---|---|---|---|
| **전체 백업** | 모든 데이터 통째로 백업 | 복구 빠름 | 시간/용량 많이 필요 |
| **증분 백업** | 마지막 백업 이후 변경분만 | 용량 적고 빠름 | 복구 시 여러 백업본 필요 |
| **차등 백업** | 마지막 전체 백업 이후 변경분 전부 | 복구 간단 | 증분보다 용량 큼 |

> 백업 데이터도 원본과 동일한 민감 정보를 담고 있으므로 **AES-256 암호화 + 별도 키 관리** 필수

## 11. SSH 실습 복습

```
VirtualBox 호스트 전용 네트워크 추가
→ Windows ↔ Ubuntu 같은 네트워크 대역에서 통신 가능
↓
sudo su -
→ root 권한 전환
↓
apt update
→ 패키지 목록 최신화
↓
apt install openssh-server
→ SSH 서버 설치. Ubuntu가 원격 접속을 받을 수 있게 됨
↓
ip a
→ Ubuntu IP 확인 (192.168.56.xxx)
↓
systemctl status ssh
→ SSH 서버 동작 확인. 포트 22번 대기 중 확인
↓
PuTTY에서 해당 IP:22로 접속
→ Windows에서 Ubuntu 터미널 원격 조종
```

| 프로토콜 | 포트 | 암호화 | 현재 사용 |
|---|---|---|---|
| Telnet | 23 | ❌ 평문 | 사용 금지 |
| **SSH** | 22 | ✅ 암호화 | 현재 표준 |
