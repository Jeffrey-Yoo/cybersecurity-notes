# 2026-06-12 복습 — 과정2 9일차: 삼바·내부DNS·NAT 포트포워딩·Alias

> 6/11에 가른 VLAN(10/20)에 **삼바 파일서버(VLAN30)·내부 DNS(VLAN40)를 더 올려 4칸을 채우고**, "어떤 망→어떤 망을 왜 열고 닫나"를 방화벽으로 박기 시작한 날. 핵심은 ① 삼바 오타 2종(testparm으로 거르기), ② BIND `allow-recursion` 내부망 한정(오픈 리졸버 차단), ③ NAT 포트포워딩(DNAT) — Dest.Address는 WAN IP, ④ ssh.socket이 sshd_config를 덮어쓰는 우분투 22.04+ 함정, ⑤ **Alias로 관리 포인트를 하나로** 줄이기.

## 오늘의 복습 퀴즈

**Q1.** [단답형] 삼바 `smb.conf`에서 오늘 밟은 오타 2개는? 이걸 적용 전에 걸러주는 명령은?

**Q2.** [흐름 이해] BIND9에서 `allow-query`는 any로 열어도 `allow-recursion`은 내부망(10.10.0.0/16)만 허용한다. 왜 재귀만 막나?

**Q3.** [실무 적용] 본컴(192.168.1.83)에서 내부 web 서버(10.10.10.10:80)에 직접 붙으려 NAT 포트포워딩을 만든다. `Dest. Address`에 뭘 넣어야 하나? 흔한 실수는?

**Q4.** [FIMS 연결] 같은 관리 IP를 WAN 룰 6개에 박았다가 Alias로 바꿨다. 이 "관리 포인트 줄이기"를 프로그래밍 개념에 빗대면?

**Q5.** [면접 대비] "sshd_config에서 Port를 바꿨는데 SSH 포트가 안 바뀐다"는 우분투 최신 서버. 원인과 해결은?

<details>
<summary>정답 보기</summary>

**A1.** `browsable`→**`browseable`**(e 들어감), `valid user`→**`valid users`**(복수형). 오타가 나도 서비스는 떠서 동작만 이상해진다 → **`testparm`**으로 Unknown parameter를 거른 뒤 `systemctl restart smbd`. (netplan generate, named-checkconf와 같은 "적용 전 문법검사" 습관)

**A2.** **재귀(recursion)를 누구에게나 열면 "오픈 리졸버"가 되어 DNS 증폭 DDoS의 발판으로 악용**되기 때문. 질의(allow-query)는 "우리 존 레코드 알려줘"라 열어도 되지만, 재귀는 "남 대신 외부 도메인까지 다 찾아줘"라 외부에 열면 공격 도구가 된다. 그래서 재귀는 내부망만. (방화벽 "최소허용" 원칙의 DNS판)

**A3.** **pfSense의 WAN IP(192.168.1.82)** — 트래픽이 "들어오는 입구" 기준. 흔한 실수는 보낸 사람인 본컴 IP(192.168.1.83)를 넣는 것. 출처를 거르고 싶으면 그건 Source 쪽에 넣는다. (NAT 룰을 만들면 WAN 허용룰은 자동 생성 체크로 같이 생김 — NAT와 Filter는 별개 단계)

**A4.** **매직넘버를 상수 하나로 빼는 것**과 같다. IP를 룰마다 하드코딩(반복)하면 바뀔 때 전부 수정 → Alias라는 "상수"로 묶으면 한 곳만 고쳐 전체 반영. 관리 포인트를 줄이는 게 방화벽 운영의 기본. (FIMS에서 socat 포트·경로를 한 곳에서 관리했던 발상과 같은 결)

**A5.** 우분투 22.04+는 SSH가 **소켓 액티베이션(`ssh.socket`)**으로 떠서 포트를 소켓이 관리 → sshd_config의 Port가 무시된다. 해결: `systemctl edit ssh.socket`으로 `ListenStream`을 새 포트로 바꾸고 `daemon-reload`+`restart ssh.socket`, 또는 `systemctl disable --now ssh.socket; systemctl enable --now ssh`로 전통 방식 전환. 확인은 `ss -tlnp | grep ssh`.

## 예상 면접 질문

**Q. 내부 DNS 서버를 운영할 때 보안상 반드시 챙기는 설정은?**
✅ 좋은 답변: "**오픈 리졸버가 되지 않게 재귀(recursion)를 내부망으로 제한**합니다. `allow-query`는 열어도 `allow-recursion`은 신뢰 대역만 허용해서, 외부가 우리 DNS를 DNS 증폭 DDoS의 발판으로 못 쓰게 합니다. 더해서 존 변경 때 **Serial을 반드시 +1**, 적용 전 `named-checkconf`/`named-checkzone`으로 문법검사, 무중단 반영은 `rndc reload`를 씁니다."
❌ 주의: "DNS 깔고 레코드 넣으면 된다"로 끝내면 약함. **오픈 리졸버 차단(allow-recursion)**과 **이름풀이≠통신(방화벽은 별개)**를 짚어야 함.

**Q. 외부에서 내부 서버로 접근을 열어줄 때 NAT를 어떻게 쓰나요?**
✅ 좋은 답변: "**DNAT(목적지 NAT) 포트포워딩**으로 방화벽 WAN 포트를 내부 서버 IP·포트로 꽂습니다. 입구(Dest.Address)는 방화벽 WAN IP 기준이고, NAT는 주소 변환일 뿐이라 **Filter 허용룰을 따로(또는 자동생성 체크로) 둬야** 실제로 통과합니다. 출처는 관리 IP로 좁혀 공격 표면을 줄이고, 반복되는 IP는 **Alias**로 묶어 운영합니다."
❌ 주의: NAT와 방화벽 룰을 같은 것으로 뭉뚱그리면 약함. **주소 변환(NAT) ≠ 통과 허용(Filter)** 2단계를 구분해야 함.

**Q. 방화벽 룰이 수백·수천 개가 되면 어떻게 관리하나요?**
✅ 좋은 답변: "**Alias로 IP·포트 그룹을 별칭화**해서 관리 포인트를 줄입니다. 관리자 IP 묶음(ADMIN_HOSTS), 포트 묶음(WEB_PORTS), 내부 대역(INTERNAL_NETS)처럼요. 그러면 IP 하나 바뀔 때 룰을 전부 손대지 않고 Alias 한 곳만 고치면 됩니다. 룰 순서(Top-down, first match)와 함께 **유지보수성**을 설계하는 게 운영의 핵심입니다."
❌ 주의: "룰을 잘 정리한다" 수준이면 약함. **Alias(중복 제거) + 룰 순서(성능·우선순위)**를 같이 말해야 함.

## 오늘 헷갈렸던 것 / 짚어둘 것

- **삼바 오타 2종** — browsable→browse**a**ble, valid user→valid user**s**. testparm으로 거르고 restart.
- **fstab엔 `-o` 없음** — 필드=공백, 옵션=콤마. netplan(YAML 들여쓰기=문법)·smb.conf(INI 들여쓰기=가독성)·fstab(공백 구분) 파서별로 규칙이 다르다.
- **BIND**: allow-recursion any = 오픈 리졸버(위험). Serial 수정 때마다 +1. FQDN 끝 점(.) 빠뜨리면 존 이름이 자동으로 또 붙음.
- **NAT 포워딩 Dest.Address = 방화벽 WAN IP**, 보낸 사람 아님. NAT(변환)와 Filter(허용)는 별개 단계.
- **ssh.socket이 sshd_config를 덮어쓴다**(우분투 22.04+). 포트 안 먹으면 `ss -tlnp`로 누가 잡았나 확인.
- **이름풀이 OK ≠ 통신 OK** — dig 되어도 방화벽이 막으면 접속 안 됨. 트러블슈팅 때 DNS와 방화벽을 분리해서 본다.
- **PHP 미설치면 PHP 코드가 그대로 출력** — 아파치가 .php를 실행 안 하고 텍스트로 뱉음. `php -v`로 확인.
