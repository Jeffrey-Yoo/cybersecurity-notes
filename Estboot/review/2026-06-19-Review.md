# 2026-06-19 복습 — Kali 공격(nmap/Hydra) · ESM · IPv4 헤더 분석

> 오늘 핵심: ① **Kali 공격 가동** — nmap 포트스캔(열린 포워딩 포트만 잡힘) + **Hydra 사전공격**(SSH 무차별 대입, 실패 로그 주체는 관리자 아니면 공격자뿐) ② **IPS 인라인+전체룰에 nmap → 알람 폭주 → ESM**(로그 중앙 수집·상관분석)의 필요성 ③ **보안관제(보안운영)의 두 축**과 솔루션 카탈로그(APC/SafePC/WebKeeper/APT(FireEye)/DDX/NAC/NDLP…) ④ **IPv4 헤더 분해** — DSCP·ECN·Total Length·Identification·MF/Offset(단편화)·TTL·Protocol·Checksum.

## 오늘의 복습 퀴즈

Q1. (단답) nmap 포트스캔에서 **열린 포트만 결과에 뜨는 이유**는? 그리고 그 "열린 포트"는 우리 랩에서 무엇 때문에 생겼나?

Q2. (흐름) Hydra가 하는 건 순수 브루트포스가 아니라 **사전공격(Dictionary)**이다 — 차이는? 그리고 서버 입장에서 "비밀번호 실패 로그 급증 = 거의 무조건 공격"이라 단정하는 근거는?

Q3. (실무) IPS 한 대에 nmap 한 번 쐈는데 알람이 쏟아졌다. 여기서 **ESM이 왜 필요해지나?** "장비 한 대로도 이런데…"로 시작해서 설명.

Q4. (정정) 수업노트의 "Identification = 네트워크가 보낼 수 있는 최대전송 단위"는 틀렸다. **Identification**과 **MTU**는 각각 무엇인가?

Q5. (면접/연결) **단편화 기반 DoS(MF=1만 계속 보내기)**와 **SYN Flood**가 "같은 원리"라는데, 그 공통 원리는?

<details>
<summary>정답 보기</summary>

**A1.** 닫힌 포트는 응답이 없거나 RST를 돌려줘서 스캔이 "열림"으로 표시하지 않는다 → 열린 포트만 결과에 남는다. 우리 랩에서 그 열린 포트는 **6/12에 박은 NAT 포트포워딩(DNAT)** 으로 외부에 노출시킨 포트들(WAN 80/1010/2020…)이다. **연 포트 = 공격 표면**이라, 안 쓰는 포워딩을 줄이면 스캔에 잡히는 표면도 준다.

**A2.** 순수 브루트포스는 가능한 모든 조합(0000,0001…)을 전부 시도하고, **사전공격**은 "자주 쓰는 패스워드 목록(wordlist)" 을 대입한다 — Hydra가 wordlist 기반이라 사전공격이 정확한 표현(노트의 "디렉터리 공격"은 딕셔너리의 오기). 실패 로그를 만드는 주체는 ① 오타 낸 진짜 관리자 ② 크리덴셜 노린 공격자, **둘뿐**이라 무더기 실패는 거의 공격으로 본다.

**A3.** 장비 한 대(IPS)에서만도 알람이 쏟아지는데, 방화벽·IDS·DLP·EDR·프록시가 각자 로그를 뱉으면 사람이 못 따라간다. 게다가 브루트포스는 "방화벽 세션 급증 + 웹서버 인증실패 급증"을 **같이 봐야** 판단된다(단일 로그론 모름). 그래서 흩어진 로그를 한곳에 모아 **수집·관리·상관분석**하는 **ESM(Enterprise Security Management)** 이 필요하다 — Security Onion을 전사 규모로 확장한 게 ESM.

**A4.** **Identification** = 한 데이터를 여러 조각으로 쪼갰을 때 "이 조각들은 한 묶음"이라고 표시하는 **식별자**(조각이 뒤섞여 와도 같은 ID끼리 모아 offset 순서로 재조립). **MTU** = 한 번에 보낼 수 있는 **최대 크기**(이더넷 보통 1500바이트). 데이터가 MTU를 넘으면 단편화되고, 그 단편들을 묶는 ID가 Identification. 완전히 다른 필드.

**A5.** 공통 원리 = **"끝맺지 않는 미완성"으로 상대 자원을 잠근다.** SYN Flood는 SYN만 보내고 ACK를 안 줘 half-open 연결이 세션 테이블을 채우고, 단편화 DoS는 MF=1 조각만 보내고 마지막(MF=0)을 안 보내 받는 쪽이 버퍼에 쌓아둔 채 무한 대기 → 둘 다 트래픽이 많지 않아도 자원이 마른다. 방어는 IPS·방화벽이 **불완전 세션·비정상 단편을 임계치로 끊는** 것.

## 예상 면접 질문

**Q. ESM(통합보안관제)이 왜 필요한가요? 장비마다 로그를 보면 되지 않나요?**
✅ 좋은 답변: "보안장비는 각자 일부만 봅니다. 방화벽은 세션, IPS는 시그니처, 웹서버는 인증 로그만 보죠. 그런데 브루트포스 같은 공격은 단일 로그가 아니라 **여러 로그의 상관관계**(세션 급증 + 인증실패 급증)를 엮어야 판단됩니다. ESM은 흩어진 로그를 한곳에 모아 **수집·정규화·상관분석**해서, 개별 장비론 못 보는 '하나의 사건'을 꿰어냅니다. 오픈소스로 치면 Security Onion의 SOC 콘솔이 같은 발상이고, ESM은 그걸 전사 규모로 확장한 것입니다."
❌ 주의: "ESM은 로그를 모아서 보관하는 저장소다"라고만 하면 부족 — 핵심은 **상관분석(correlation)**.

**Q. IDS/IPS에서 시그니처를 짤 때 content를 그냥 문자열로 넣으면 왜 부족한가요? (또는: 포트스캔/Hydra 공격은 무슨 로그로 잡나요?)**
✅ 좋은 답변: "공격은 인코딩·대소문자·비출력 바이트로 같은 글자도 바이트가 달라져 평문 매칭이 빗나갑니다. 그래서 hex로 바이트를 그대로 박거나 파일 해시로 잡습니다. 포트스캔은 IPS 알람·방화벽 세션으로, Hydra 무차별 대입은 서버 **auth 실패 로그 급증 + 세션 급증**의 상관관계로 잡고, 차단은 fail2ban·Rate Limit·IPS 임계치로 합니다."
❌ 주의: "포트스캔은 nmap으로 잡는다"는 공수 혼동 — nmap은 **공격 도구**지 탐지 수단이 아니다.

**Q. TCP SYN Flood와 IP 단편화 DoS의 공통점은?**
✅ 좋은 답변: "둘 다 **연결/조각을 일부러 끝맺지 않아** 받는 쪽이 무한 대기하며 자원을 점유하게 만드는 자원 고갈 공격입니다. SYN Flood는 half-open 세션으로 세션 테이블을, 단편화 DoS는 미완성 단편으로 재조립 버퍼를 채웁니다. 트래픽 양으로 압도하는 플러딩과 달리 적은 양으로도 자원을 마르게 합니다."
❌ 주의: 모든 DoS를 "트래픽 폭탄"으로만 설명하면 이 유형(half-open류)을 놓친다.
