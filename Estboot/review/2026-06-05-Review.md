# 2026-06-05 복습 — 과정2 4일차: HA(이중화) & IPS·웹방화벽 & 탐지 패러다임(시그니처 vs 애노멀리)

> 오늘 핵심은 두 덩어리. ① 장비를 어떻게 이중화하나(HA — Active-Standby / Active-Active / 본딩·티밍) ② **공격을 어떻게 탐지하나**(시그니처 기반 ↔ 애노멀리 디텍션 → 동적분석 샌드박스 → Darktrace AI). 졸려서 놓쳤다던 마지막 슬라이드(161416)가 사실상 오늘의 결론.

## 오늘의 복습 퀴즈

**Q1.** [단답형] Active-Standby에서 주 장치와 보조 장치를 **동일 성능**으로 맞춰야 하는 이유는?

**Q2.** [흐름 이해] 시그니처 기반 탐지와 애노멀리 디텍션의 결정적 차이는? 각각 못 잡는 게 뭔가?

**Q3.** [실무 적용] 시그니처가 아직 없는 신규 악성코드 의심 파일이 들어왔다. 이걸 어떻게 분석하나? 왜 VM을 쓰나?

**Q4.** [FIMS 연결] 아키텍처 그림에서 라인 옆에 빠져있던 "분석" 박스, 그리고 Darktrace의 "분석 모드" — FIMS의 어떤 구성요소가 이 "분석/관제"의 초기형인가?

**Q5.** [면접 대비] "DDoS는 IPS로도 막을 수 있나요?"라고 물으면?

<details>
<summary>정답 보기</summary>

**A1.** 세션을 동기화(클러스터)해서 주 장치가 들고 있던 **세션 상태를 그대로 이어받아야** 하기 때문. 성능이 다르면 failover로 트래픽을 넘겨받는 순간 스루풋을 못 버텨 보조도 같이 뻗는다. (주가 죽으면 VIP를 보조로 넘겨 사용자는 같은 VIP로 무중단 접속)

**A2.** 시그니처 = **알려진** 공격의 패턴(흔적·코드 문자열·공격 순서)을 DB와 대조 → 처음 보는/변종·난독화 공격은 못 잡음. 애노멀리 = **정상 기준선(baseline)을 학습**해두고 거기서 벗어나면 의심 → 신규 공격도 잡지만 정상을 잘못 잡는 **오탐(false positive)**이 약점. 둘은 대체가 아니라 보완 관계.

**A3.** **동적 분석** — 의심 파일을 **격리된 VM 샌드박스**에서 실제로 실행시켜 행동(파일 생성, 레지스트리 변조, 외부 C2 통신 등)을 관찰한다. 진짜로 실행해야 행동이 드러나는데 운영 서버에서 실행하면 그게 곧 감염이라, 언제든 갈아엎을 수 있는 VM에서 터뜨려보고 끝나면 스냅샷으로 초기화한다. (= Anti-APT 샌드박스 = "동적분석기 → vm")

**A4.** **Wazuh(관제 서버)**. 여러 장비 로그를 모아 연관분석한 게 "분석/관제"의 초기형. 여기에 자가학습 AI가 붙어 사람이 룰을 안 짜도 평소와 다른 걸 알아서 잡으면 Darktrace 방향이 된다.

**A5.** "전문 DDoS 장비만큼은 아니지만 가능합니다. IPS에 임계치(Threshold)를 걸어 — 예를 들어 SYN Proxy 초당 500, CPS Flood 초당 100, Connection Flood 동시연결 25 같은 식으로 — 플러딩을 1차 방어합니다. 다만 IPS 자체에 부하가 걸리면 뻗을 수 있어, 뻗었을 때의 대응 체계까지 같이 설계해야 합니다."

## 예상 면접 질문

**Q. 시그니처 기반 탐지의 한계와, 그걸 보완하는 방식을 설명하라.**
✅ 좋은 답변: "시그니처는 알려진 공격의 패턴을 DB와 대조하는 방식이라 빠르고 오탐이 적지만, DB에 없는 신규 공격이나 난독화된 변종은 못 잡습니다. 이를 보완하는 게 애노멀리 디텍션입니다. 정상 트래픽을 먼저 학습해 기준선을 만들고 거기서 벗어나는 걸 잡기 때문에 제로데이도 탐지할 수 있습니다. 대신 오탐 관리가 핵심 과제이고, Darktrace처럼 내부를 통째로 자가학습하는 AI 기반 NDR로 발전하고 있습니다."
❌ 주의: "시그니처는 구식, 애노멀리가 우월" 식의 우열 구도로 말하면 안 됨. **대체가 아니라 보완 관계** — 아는 공격은 시그니처가 빠르고 정확하다.

**Q. 샌드박스(Anti-APT)는 어떻게 악성코드를 잡나?**
✅ 좋은 답변: "동적 분석 방식입니다. 의심 파일을 격리된 VM에서 실제로 실행시켜, 파일 생성·레지스트리 변조·외부 통신 같은 악성 행위가 나타나는지 관찰합니다. 시그니처가 없어도 행위로 잡을 수 있어 제로데이 대응이 가능하고, 잡으면 그 특징을 시그니처로 추출해 다른 장비·백신에 배포합니다. EDR의 행위 기반 탐지와 같은 발상입니다."
❌ 주의: "샌드박스는 파일을 검사한다"처럼 뭉뚱그리면 약함. **실행해서 행동을 본다(동적)** vs 코드만 본다(정적)의 구분이 핵심.

**Q. Active-Standby와 Active-Active의 차이는?**
✅ 좋은 답변: "Active-Standby는 주 장치만 일하고 보조는 세션을 동기화받으며 대기하다가, 주가 죽으면 VIP를 넘겨받아(failover) 무중단으로 잇습니다. Active-Active는 두 대가 동시에 부하를 나눠 처리하며 세션을 서로 동기화하는데, 양쪽 라인을 다 깔아야 해서 비용이 큽니다. 공통으로 세션 동기화 때문에 동일 성능 장비로 맞춰야 합니다."
❌ 주의: "Active-Active가 항상 좋다"고 하면 안 됨. **비용 트레이드오프**(라인 이중 구축, 풀 메쉬)를 같이 말해야 함.

## 연결되는 기존 개념

| 오늘 개념 | 연결되는 기존 개념 | 파일 | 어떻게 연결되나 |
|---|---|---|---|
| HA — VIP failover | VIP, 세션테이블, 로드밸런서 | OSI-7Layer / Architecture | 주 장치가 죽으면 VIP를 보조로 넘겨 같은 가상IP로 무중단 |
| HA 디도스 예외 | DDoS, 스루풋 100% 초과 | Security-Solutions | 공격으로 스루풋 초과한 건 failover 대상이 아님(넘겨도 같이 뻗음) |
| 본딩·티밍 | 회선 이중화, 고가용성 | Architecture | Active-Active의 "양쪽 라인 구현"이 NIC 묶기로 구체화 |
| TP모드 한계(망제어·확장 불가) | 1세대 ACL≠망분리, TP모드 | Security-Solutions | 망을 나누는 주체가 아니라 1차 방화벽 대체 불가 |
| 룰 처리 순서(Top-Down) | UFW 룰 순서, 방화벽 정책 철칙 | Security-Solutions | 첫 매칭에서 멈추니 차단룰은 최상단, 트래픽 많은 룰은 위로 |
| 커스텀 시그니처 GUI | 시그니처란, 커스텀 시그니처 | Security-Solutions | content로 페이로드 문자열 매칭 → NGFW 애플리케이션 식별과 같은 결 |
| IPS DDoS 임계치 | DDoS 방어, Threshold | Security-Solutions | SYN/CPS/Connection Flood 임계치 = 애노멀리 Threshold의 실제값 |
| 시그니처 vs 애노멀리 | 시그니처란, EDR 행위탐지 | Security-Solutions | 시그니처=아는 공격 / 애노멀리=평소와의 차이, 보완 관계 |
| 동적분석 샌드박스(VM) | Anti-APT, EDR, "분석" 박스 | Security-Solutions | 의심 파일을 VM에서 실행해 행동 관찰 = 아키텍처의 분석 장비 |
| Darktrace(자가학습 AI) | 관제(SIEM) 연관분석, 애노멀리 | Security-Solutions | 관제+행위탐지+애노멀리가 AI로 합쳐지는 방향, Wazuh가 초기형 |

## 오늘 헷갈렸던 것 / 짚어둘 것

- **161416 슬라이드 전체를 졸려서 놓쳤던 것** — 사실 오늘의 결론.
  - 정리: 슬라이드는 **탐지의 두 패러다임**을 보여준 것. 시그니처 기반(아는 공격 대조) ↔ 애노멀리 디텍션(학습모드→통계→오탐관리→패턴 재정의) + Threshold(임계치). → Security-Solutions.md "탐지의 두 가지 패러다임"에 정리.

- **"동적분석기 → vm"이 뭔 소린지 몰랐던 것**
  - 정리: 동적 분석 = 의심 파일을 **격리된 VM(샌드박스)에서 실제 실행**해 행동을 보는 것. Anti-APT가 바로 이것. "내부 보안서버에 악성코드 유입 → 동적분석기가 VM에서 터뜨려봄 → 악성이면 차단+시그니처 추출". → Security-Solutions.md Anti-APT "정적/동적 분석"에 정리.

- **아키텍처 그림에서 라인 옆 "분석" 박스의 정체**
  - 정리: 그 분석 장비 = 동적분석 샌드박스. 라인에서 의심 파일을 떠다 VM에서 실행해보는 곳. IDS(미러링)처럼 라인 밖에 빠져있던 이유와 같은 결.

- **Darktrace가 그냥 "AI 보안"으로만 들렸던 것**
  - 정리: 시그니처를 안 쓰고 **내부 전체를 자가학습**해 평소와 다른 걸 잡는 NDR. 사람이 룰을 안 짜도 됨 = 애노멀리 디텍션의 AI 본격화. 탐지 모드(실시간 차단) vs 분석 모드(사후 수사) 두 갈래. "앞으로 가야 할 방향".

- **초반(L3-only / VLAN / L2 방화벽)은 복습성** — L3-only는 1세대 한계(ACL≠망분리), L2(TP모드)는 IP 없음·바이패스 필요·망확장/내부망제어 불가 → 1차 방화벽 대체 불가. Security-Solutions.md "TP모드의 한계"에 보강.
