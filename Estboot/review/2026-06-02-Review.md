# 2026-06-02 복습 — 과정2 2일차: 보안 장비 배치 전략 & L3/L4/L7 심화

> 오늘 핵심은 "어떤 보안 장비를, 왜 그 순서로, 어디에 두느냐". 거기에 ARP·세션테이블·VIP·인라인/바이패스까지 묶음.

## 오늘의 복습 퀴즈

**Q1.** [단답형] 보안 장비를 트래픽 진입 순서대로 나열하면? (Anti-DDoS부터)

**Q2.** [흐름 이해] IDS는 왜 인라인이 아니라 미러링으로 빼서 운용하냐?

**Q3.** [실무 적용] 웹 서버만 있는 서비스인데 IPS, WAF, Anti-APT를 다 넣자는 요청이 왔다. 어떻게 판단하겠는가?

**Q4.** [FIMS 연결] FIMS의 Suricata와 Wazuh는 오늘 배운 장비 중 각각 무슨 역할에 해당하냐?

**Q5.** [면접 대비] L4 방화벽만으로는 못 막는 공격이 있다. 무엇이고 왜인가?

<details>
<summary>정답 보기</summary>

**A1.** Anti-DDoS → 방화벽(L4) → IPS → Anti-APT → WAF → L3 라우터 → [IDS는 미러링으로 옆에] → 서버. (큰 것부터 거르고 점점 정밀하게)

**A2.** IDS는 차단 안 하는 로깅 장비라 라인 위에 둘 이유가 없음. 인라인에 넣으면 장애 시 라인 절체로 다운타임 발생 → 복제(미러링) 데이터만 받아서 분석하면 넣었다 뺐다 자유롭고 서비스 영향 없음.

**A3.** "틀린 아키텍처는 없고 요건에 맞는지가 핵심". 웹 서버만 있으면 WAF 하나로 충분할 수 있음. 소켓 서버(메신저 등)가 같이 있으면 IPS 추가. Anti-APT는 신규 악성코드 위협이 큰 환경에서. 무조건 다 넣는 건 비용·동맥경화만 유발.

**A4.** Suricata = IPS 역할(인라인, 시그니처 기반 차단). Wazuh = 관제 서버 역할(여러 로그 수집 + 연관 분석 = SIEM 개념).

**A5.** IP를 변조(위조)해서 들어오는 공격. L4는 IP+포트만 보고 페이로드(내용)를 안 보기 때문에 변조된 IP를 못 잡음. 내용까지 보려면 L7(WAF/NGFW)이 필요.

## 예상 면접 질문

**Q. 보안 장비 배치 순서를 설명하고, 왜 그 순서인지 근거를 말하라.**
✅ 좋은 답변: "Anti-DDoS를 맨 앞에 둡니다. 대량 트래픽을 먼저 걸러야 뒤의 방화벽·IPS가 안 죽기 때문입니다. 이후 방화벽(L4 접근제어) → IPS(시그니처 차단) → Anti-APT(샌드박스 신규 탐지) → WAF(HTTP 정밀) 순으로 점점 정밀하게 검사합니다. IDS는 차단을 안 하므로 인라인 대신 미러링으로 옆에 둡니다."
❌ 주의: "방화벽이 제일 앞"이라고 하면 안 됨. DDoS 트래픽이 방화벽을 먼저 죽이기 때문에 Anti-DDoS가 더 앞이다.

**Q. IPS와 WAF가 같이 있으면 중복 아닌가?**
✅ 좋은 답변: "아닙니다. IPS는 모든 프로토콜을 대상으로 하고 WAF는 HTTP 전용 전문 장비입니다. 태생이 다릅니다. 웹 서버만 있으면 WAF로 충분하지만, 소켓 서버 같은 비-HTTP 서비스가 함께 있으면 IPS가 필요합니다."
❌ 주의: "둘 다 공격 막으니 하나만 있으면 된다"는 틀림. 담당하는 트래픽 범위가 다르다.

**Q. 인라인 장비를 점검·교체할 때 서비스 중단 없이 하려면?**
✅ 좋은 답변: "하드웨어 바이패스를 활용합니다. 장비 내장 바이패스는 전원이 꺼지면 자동으로 입출력을 직결해 트래픽을 통과시킵니다. 계획된 교체·점검에는 외부 바이패스 장비로 먼저 우회 전환한 뒤 인라인 장비를 빼면 서비스 무중단으로 작업할 수 있습니다."
❌ 주의: "장비 빼면 잠깐 끊긴다"로 끝내면 안 됨. 바이패스로 무중단 처리하는 게 실무.

## 오늘 헷갈렸던 것 / 짚어둘 것

- **인라인 vs 미러링** — "IPS랑 IDS 둘 다 패킷 보는데 왜 위치가 다르지?"
  - 정리: 보는 건 둘 다 L7까지지만, **차단하느냐(인라인)** vs **로그만 남기느냐(미러링)**가 갈림. 차단 장비는 길목에 서 있어야 하고, 로깅 장비는 옆에서 복제만 받으면 됨.
  - 비유: 검문소(IPS, 차 세움) vs 길가 CCTV(IDS, 지나가는 거 녹화).

- **VIP가 실제 IP인 줄 알았던 것**
  - 정리: 사용자가 접속하는 건 VIP(가상 IP). 방화벽이 DNAT으로 실제 서버 IP에 1:1 매핑하고, L4가 그 하위 서버들에 로드밸런싱. nslookup으로 보이는 건 보통 VIP다.

- **"틀린 아키텍처"를 찾으려던 습관**
  - 정리: 틀린 아키텍처는 없다. "개선·최적화할 게 있나"로 봐야 함. 요건(서비스 특성·트래픽량)에 맞는지가 기준. 작은 처리량 장비를 큰 구간에 넣으면 동맥경화.
