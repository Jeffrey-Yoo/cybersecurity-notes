# 2026-06-15 복습 — 과정2 10일차: 웹 프록시(Squid/SquidGuard) + Snort IDS

> 들어오는 것(인바운드 NAT·방화벽)만 보던 데서 **나가는 것(아웃바운드 프록시)**과 **흐르는 것(IDS 가시성)**까지 손댄 날. 핵심은 ① 포워드 프록시로 **아웃바운드 URL 통제**(web=화이트/webdev=블랙), ② HTTPS는 SSL Bump가 필요한데 투명 intercept는 SNI 소실로 깨짐 → 명시적 프록시가 정석, ③ **DNS는 80/443 불필요**(DNS≠게이트웨이), ④ Snort는 ET Open(무료) 룰로 4종 탐지, ⑤ **ET Open엔 범용 SQLi가 없다 → IDS≠WAF**, ⑥ threshold 때문에 같은 IP론 재탐지가 안 된다, ⑦ "GUI 초록불 ≠ 프로세스 실행"(바이너리 미설치).

## 오늘의 복습 퀴즈

**Q1.** [단답형] 포워드(웹) 프록시를 쓰는 보안상 이유 3가지는? 그리고 "가장 좋은 방법"은 따로 무엇인가?

**Q2.** [흐름 이해] HTTP는 프록시가 URL을 그냥 거르는데 HTTPS는 못 거른다. 거르려면 뭐가 필요하고, 투명 프록시로 했을 때 왜 `ERR_DNS_FAIL`이 났나?

**Q3.** [개념 구분] DNS 서버(VLAN40)의 아웃바운드 80/443을 닫았다. "DNS도 인터넷을 쓰는데 막아도 되나?"라는 질문에 어떻게 답하나?

**Q4.** [실무 적용] Snort에 ET Open 룰을 다 켰는데 `' OR '1'='1` SQLi가 안 잡힌다. 왜 그런가? 이건 어느 장비가 담당하나?

**Q5.** [면접/삽질] C2 탐지 룰이 한 번 뜨고 다시 안 뜬다. 룰이 죽은 건가? 그리고 "Snort GUI는 초록불인데 프로세스가 안 떴다"의 원인은?

**A1.** ① 악성 사이트 사전 차단, ② 서버의 무분별한 인터넷 접근 차단(감염 예방·보안정책 유지 — 감염 서버가 C2로 나가려 해도 막힘), ③ 침해 대응 시 가시성 확보(로그). **가장 좋은 방법은 "필요한 목적지만 1:1로 여는 것"** — 프록시는 그게 현실적으로 안 될 때(보안 인력 부족, outbound=any) 아웃바운드를 한 곳에 모아 거르는 **차선책**이다.

**A2.** **SSL Bump**(방화벽이 CA 인증서를 내주고 중간에서 복호화)가 필요하다. 투명 intercept는 클라이언트가 `CONNECT 도메인:443`을 안 보내고 **ClientHello의 SNI(접속 도메인)에 의존**하는데, SNI가 소실/누락되면 Squid가 어느 도메인인지 못 찾아 DNS 해석에 실패 → `ERR_DNS_FAIL`. 그래서 HTTPS 필터링은 **명시적 프록시(PAC/WPAD, 클라이언트가 도메인을 CONNECT로 알려줌)**가 정석. (+ 모든 HTTPS를 다 복호화하는 게 정답도 아님 — 자원 낭비·프라이버시)

**A3.** 막아도 된다. **DNS는 53번으로 "이름 안내"만** 한다(외부 도메인 포워딩도 8.8.8.8:53). 실제 웹 트래픽(80/443)은 **각 서버가 게이트웨이를 통해 직접** 나가는 것이지 DNS를 경유하는 게 아니다. **DNS ≠ 게이트웨이** — DNS는 주소 안내, 라우팅은 pfSense가 한다. 완전히 별개 레이어. (6/12 "이름풀이 OK ≠ 통신 OK"의 연장)

**A4.** **ET Open엔 `' OR '1'='1` 같은 범용 SQLi 패턴이 의도적으로 빠져 있다**(오탐이 너무 많아서). ET Open은 알려진 CVE·특정 제품 기반 시그니처 위주라, 범용 SQLi는 커스텀 룰(`content:"pw="; content:"OR"; distance:0; within:50;`)을 직접 짜야 한다. **범용 웹 공격(SQLi/XSS)은 WAF(ModSecurity+OWASP CRS) 영역** — IDS ≠ WAF, 역할이 다르다.

**A5.** 룰이 죽은 게 아니라 **threshold**다. `track by_src, count 1, seconds 3600`이면 한 소스에서 1시간에 1번만 alert → 다른 소스 IP로 쳐야 재현된다. // "GUI 초록불인데 프로세스 안 뜸"은 **패키지 등록(메타데이터)만 되고 바이너리는 미설치**였던 것("Package is configured but not (fully) installed"). `find /usr/local -name snort -type f`로 바이너리 없음 확인 → 완전 삭제 후 재설치로 해결. 데몬 죽은 사유는 alert 로그가 아니라 **stderr**에 있다(과정1 Suricata fast.log 삽질과 동일 구조).

## 예상 면접 질문

**Q. 서버의 아웃바운드(나가는) 트래픽은 어떻게 통제하나요?**
✅ 좋은 답변: "이상적으로는 서버마다 **필요한 목적지만 1:1로** 여는 게 가장 안전합니다. 다만 인력·서버가 많은 환경에선 현실적으로 어려워, **포워드 프록시(Squid 등)나 시큐어 웹 게이트웨이를 앞단에 두고 URL 단위로 화이트/블랙리스트를 겁니다.** 감염 서버가 외부 C2로 나가는 것도 여기서 막히고, 로그로 가시성도 확보됩니다. 더 나아가 패치는 서버가 직접 인터넷에 나가는 대신 **PMS(내부 레포)를 경유**시키고, 안 쓰는 아웃바운드 포트는 닫아 공격 표면을 줄입니다."
❌ 주의: "방화벽으로 막는다"로만 끝내면 약함. 방화벽은 IP/포트 레벨, **프록시는 URL/도메인 레벨 통제**라는 층위 차이와 "1:1이 최선, 프록시는 차선"이라는 절제를 짚어야 함.

**Q. HTTPS 트래픽도 내용 기반으로 필터링하려면 어떻게 하나요? 주의점은?**
✅ 좋은 답변: "**SSL Bump / SSL Inspection**으로, 방화벽이 자체 CA 인증서를 발급해 클라이언트에 신뢰 등록시키고 중간에서 복호화해 검사합니다. 단 ① **투명 intercept 방식은 SNI 소실 등으로 불안정**해 실무에선 PAC/WPAD로 프록시를 명시 지정하는 방식을 더 쓰고, ② **모든 HTTPS를 다 복호화하는 건 자원 낭비·프라이버시 이슈**라 금융·인증·의료 같은 민감 트래픽은 복호화 예외를 둡니다. '태울지 말지'를 목적과 자원으로 판단해야 합니다."
❌ 주의: "복호화하면 다 본다"만 말하면 약함. **투명/명시적 차이**와 **선택적 복호화(예외 정책)**를 같이 말해야 함.

**Q. IDS(Snort)를 깔았는데 SQL Injection이 안 잡힙니다. 무엇을 점검하나요?**
✅ 좋은 답변: "먼저 **룰셋의 한계**를 봅니다 — ET Open 같은 무료 룰셋은 오탐 때문에 범용 SQLi 패턴을 빼두는 경우가 많아, 필요하면 **커스텀 룰**을 작성합니다. 다음 **인터페이스별 룰 활성화**(Snort는 인터페이스마다 룰이 독립)와 **threshold**(count/seconds 조건으로 재현이 안 될 수 있음)를 점검합니다. 다만 근본적으로 **범용 웹 공격 탐지·차단은 WAF의 역할**이라, IDS만으로 SQLi/XSS를 다 막으려는 설계 자체가 잘못입니다. IDS≠WAF."
❌ 주의: "룰 업데이트하면 된다"로 끝내면 약함. **IDS와 WAF의 역할 분리**가 핵심 포인트.

## 오늘 헷갈렸던 것 / 짚어둘 것

- **SquidGuard는 Save+Apply** — Apply 따로 안 누르면 적용 안 됨.
- **투명 HTTPS(SSL Bump)는 SNI 소실로 ERR_DNS_FAIL** — 명시적 프록시(PAC/WPAD)가 정석.
- **"TP모드" 두 개 혼동 금지** — Squid 투명 프록시(L7) vs 방화벽 TP모드(L2 브릿지).
- **DNS≠게이트웨이** — DNS는 53번 이름안내만, 트래픽 운반은 pfSense가. 80/443 불필요.
- **Snort GUI 초록불 ≠ 프로세스 실행** — 바이너리 미설치(등록만 됨). `find`로 바이너리, `-T`로 설정 파싱 확인.
- **죽은 사유는 alert 로그 아닌 stderr** — pfSense 셸은 sh라 `2>&1` 안 먹어 `2>/tmp/file`로 분리.
- **threshold 탓에 같은 IP 재테스트 안 됨** — `count 1`이면 다른 소스로, `count 5`면 5회 이상.
- **ET Open엔 범용 SQLi 없음** — 커스텀 룰 작성. 범용 웹 공격은 WAF(IDS≠WAF).
- **음차 교정**: "vgt"→**ET Open(무료 룰셋)**, "웹클라이드"→**상용 SWG**(Zscaler/Blue Coat/Forcepoint/WeBKeeper 등).
